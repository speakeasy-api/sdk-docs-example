## Pagination

Many top-level resources have support for bulk fetches via list API methods. For instance,
you can [list connections](#connections), [list directory users](#dirUsers), and [list directory groups](#dirGroups).
These list APImethods share a common structure, taking at least these four parameters.

WorkOS utilizes pagination via the <span className={'default'}>after</span> and <span className={'default'}>before</span> parameters.
Both parameters take an existing object ID value and return objects in either descending or ascending order by creation time.

```php copy filename="Request / Response"
<?php

    WorkOS\WorkOS::setApiKey("sk_example_123456789");

    $sso = new WorkOS\SSO();

    [$before, $after, $connections] = $sso->listConnections(limit: 100);

    while ($before) {
        [$before, $after, $currentPage] = $sso->listConnections(
            limit: 100,
            before: $before
        );
    $connections = array_merge($connections, $currentPage);
}
```

## Webhooks

WorkOS uses webhooks to automatically notify your application any time certain changes
are made to your connections.

A webhook URL is an endpoint hosted by your application that subscribes to notifications
from WorkOS. WorkOS sends these notifications as events which detail changes to users and groups.
For example, youâ€™ll receive an event when new users are created, group properties are updated, or users have been added to a group, etc.
